# OpenWrt Toolbox — дизайн установщика

Дата: 2026-07-23  
Версия проектируемого скрипта: 0.0.1

## Цель

Создать один самостоятельный интерактивный POSIX `sh`-скрипт для установки, удаления и проверки полезных компонентов на OpenWrt. Скрипт должен одинаково работать в системах с `opkg` и `apk`, быть понятным для дальнейшего расширения и не требовать Bash.

## Состав версии 0.0.1

1. Тема LuCI Proton 2025 из репозитория `ChesterGoodiny/luci-theme-proton2025`.
2. Веб-терминал:
   - `luci-app-ttyd`;
   - `luci-i18n-ttyd-ru`.
3. SFTP:
   - `openssh-sftp-server`.

## Репозиторий

Основной исполняемый файл:

```text
openwrt-toolbox.sh
```

Вся логика располагается в одном файле. Внутри файл разделяется комментариями и функциями на независимые смысловые блоки:

1. метаданные и версия;
2. константы;
3. оформление вывода;
4. общие проверки;
5. абстракция пакетного менеджера;
6. операции Proton 2025;
7. операции ttyd;
8. операции SFTP;
9. групповые операции;
10. универсальное меню компонента;
11. главное меню;
12. точка входа.

## Точка входа

Скрипт запускается командой из README:

```sh
wget -O /tmp/openwrt-toolbox.sh https://raw.githubusercontent.com/ang3el7z/openwrt-toolbox/main/openwrt-toolbox.sh && sh /tmp/openwrt-toolbox.sh
```

Сначала файл сохраняется, затем запускается. Это оставляет стандартный ввод свободным для интерактивного меню.

## Предварительные проверки

Перед показом меню скрипт:

1. проверяет запуск с правами `root`;
2. проверяет наличие `/etc/openwrt_release`;
3. читает отображаемую версию OpenWrt;
4. определяет пакетный менеджер по доступной команде:
   - сначала `apk`;
   - затем `opkg`;
5. завершает работу с понятной ошибкой, если менеджер не найден;
6. проверяет наличие инструмента загрузки: `wget` или `uclient-fetch`.

Определение не привязывается только к номеру версии OpenWrt: фактически доступная команда является источником истины.

## Главное меню

```text
OpenWrt Toolbox
Версия: 0.0.1
Система: OpenWrt <версия>
Менеджер пакетов: <apk|opkg>

1. Все компоненты
2. Тема Proton 2025
3. Веб-терминал ttyd
4. SFTP-сервер
0. Выход
```

Каждый компонент и раздел «Все компоненты» открывает подменю:

```text
1. Установить
2. Удалить
3. Показать состояние
0. Назад
```

Для отдельных компонентов применяется одна универсальная функция меню, получающая название и функции установки, удаления и состояния.

## Абстракция пакетного менеджера

Общие функции скрывают различия между менеджерами:

| Операция | opkg | apk |
|---|---|---|
| Обновить индекс | `opkg update` | `apk update` |
| Установить пакет из репозитория | `opkg install` | `apk add` |
| Установить локальный пакет темы | `opkg install <файл.ipk>` | `apk add --allow-untrusted <файл.apk>` |
| Удалить пакет | `opkg remove` | `apk del` |
| Проверить установку | `opkg list-installed` | `apk info -e` |

Индекс пакетов обновляется не более одного раза за один запуск скрипта.

## Proton 2025

Установка:

1. определяет требуемое расширение `.ipk` или `.apk`;
2. получает сведения об актуальном GitHub Release;
3. выбирает ровно один asset пакета `luci-theme-proton2025`;
4. прекращает установку при отсутствии или неоднозначном выборе asset;
5. загружает пакет во временный каталог;
6. устанавливает его через выбранный менеджер;
7. проверяет фактическое наличие пакета;
8. переключает LuCI на Proton 2025 через UCI после успешной установки;
9. перезапускает `uhttpd`.

Удаление:

1. переключает LuCI на Bootstrap;
2. фиксирует изменения UCI;
3. удаляет `luci-theme-proton2025`;
4. перезапускает `uhttpd`.

Скрипт не удаляет Bootstrap.

## ttyd

Установка добавляет:

- `luci-app-ttyd`;
- `luci-i18n-ttyd-ru`.

После успешной установки сервис `ttyd`, если его init-скрипт существует, включается и запускается.

Удаление удаляет только два перечисленных пакета. Системные зависимости автоматически не очищаются.

Состояние выводится отдельно для приложения и русской локализации.

## SFTP

Установка добавляет `openssh-sftp-server`. Существующий SSH-сервер Dropbear не заменяется и не удаляется.

Удаление удаляет только `openssh-sftp-server`.

Состояние определяется по наличию установленного пакета.

## Групповые операции

«Установить всё» выполняет компоненты в порядке:

1. Proton 2025;
2. ttyd;
3. SFTP.

«Удалить всё» выполняет обратный порядок:

1. SFTP;
2. ttyd;
3. Proton 2025.

Ошибка одного компонента фиксируется, но не скрывает результат остальных. После выполнения отображается сводка по каждому компоненту. Групповая операция возвращает ошибку, если хотя бы один обязательный компонент не удалось обработать.

## Повторные операции

- Установка уже установленного репозиторного пакета считается успешным состоянием и не требует принудительной переустановки.
- Удаление отсутствующего пакета считается успешным состоянием.
- Для темы пользователь может повторно запустить установку, но без отдельной функции принудительного обновления в версии 0.0.1.
- Значения меню проверяются; неизвестный ввод не выполняет действий.

## Безопасность и очистка

- Все переменные заключаются в кавычки при использовании.
- Временный каталог создаётся под `/tmp`.
- Очистка выполняется через `trap`.
- Загруженный asset должен быть обычным файлом с ожидаемым расширением.
- Команды из сетевых ответов не исполняются.
- Парсинг GitHub API ограничивается поиском URL подходящего release asset.
- Удаление не использует автоматическую очистку зависимостей.
- Скрипт не изменяет SSH-конфигурацию и пароль root.

## Ошибки и вывод

Вывод делится на уровни: информация, успех, предупреждение и ошибка. Цвет используется только при наличии терминала; без TTY вывод остаётся читаемым.

Критические предварительные ошибки завершают скрипт. Ошибка отдельного действия возвращает пользователя в меню после сообщения и не обрывает интерактивную сессию.
